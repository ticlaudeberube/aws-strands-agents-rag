# Response Cache Population Guide

## Overview

The `sync_answers_cache.py` script loads pre-generated question-answer pairs into Milvus `response_cache` collection, enabling instant semantic cache hits for known questions.

## How It Works

1. **Pre-generated Q&A pairs** stored in `data/answers.json`
2. **Question embeddings** generated using Ollama embed model
3. **Inserted into `response_cache`** collection with answers as text
4. **Semantic matching** on future queries with 92% similarity threshold

## Setup

### 1. Create Pre-Generated Answers

Edit `data/answers.json` with your Q&A pairs:

```json
{
  "qa_pairs": [
    {
      "question": "Your question here?",
      "answer": "Your detailed answer here...",
      "collection": "milvus_rag_collection"
    }
  ]
}
```

### 2. Load Answers into Cache

```bash
python document-loaders/sync_answers_cache.py
```

**Output:**
```
Loading 5 Q&A pairs into response_cache...
Generating embeddings: 100%|████████| 5/5
Inserting into response_cache...
✓ Successfully inserted 5 Q&A pairs
```

### 3. Restart API Server

```bash
python api_server.py
```

The response cache is now populated and ready for semantic matching.

## How Semantic Caching Works

**First query:**
```
User: "What is Milvus?"
→ Embedding generated
→ Milvus response_cache searched
→ Most similar pre-generated answer found (>92% similarity)
→ Returns cached answer instantly (~0ms)
```

**Similar query:**
```
User: "Tell me about Milvus"
→ Embedding generated
→ Milvus response_cache searched
→ Previous answer found (similar enough)
→ Returns cached answer instantly (~0ms)
```

**New question (not in cache):**
```
User: "How do I optimize aggregation?"
→ Embedding generated
→ No similar cached answer (< 92% similarity)
→ Full RAG pipeline executes
→ Answer generated by LLM (~24s)
→ Stored in response_cache for future similar questions
```

## Workflow

### For Development

1. **Generate answers manually** during development
2. **Add to `data/answers.json`**
3. **Run `sync_answers_cache.py`**
4. **Test with API**

### For Production

1. **Collect frequently asked questions** from users
2. **Generate high-quality answers** (AI-assisted or manual)
3. **Update `data/answers.json`**
4. **Run sync script during deployment**
5. **Monitor cache hit rates** via `/api/cache/stats` endpoint

## Monitoring

Check cache statistics:
```bash
curl http://localhost:8000/api/cache/stats
```

**Response includes:**
- `embedding_cache`: Question embedding vectors
- `search_cache`: Vector search results
- `answer_cache`: Generated answers
- `response_cache`: Persistent cached Q&A pairs

## Troubleshooting

**Error: answers.json not found**
- Create `data/answers.json` with Q&A pairs

**No Q&A pairs inserted**
- Check format of Q&A pairs
- Ensure `question` and `answer` fields exist
- Run with verbose output for debugging

**Embeddings not matching**
- Verify Ollama is running and accessible
- Check embedding model matches configuration
- Ensure vector dimensions are consistent (384 or 768)

## Performance Notes

- **Embedding generation**: ~50-100ms per question
- **Insertion into Milvus**: ~10-20ms per Q&A pair
- **Semantic search**: <1ms for cache lookups
- **Total population time**: ~1-2 seconds for 100 Q&A pairs
